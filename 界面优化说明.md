# 界面优化说明

## 1. 文档目的

这份文档用于记录当前改枪码网站的界面优化方向、已完成内容、当前实现方式，以及后续继续优化时应该优先查看和修改的位置。

当前项目路径：

- `E:\codex\guns`

当前主要前端文件：

- `index.html`
- `styles.css`
- `app.js`

---

## 2. 本轮优化目标

本轮界面优化的核心目标有四个：

1. 页面整体更简洁，不展示明显的说明型大标题和开发阶段文案
2. 整站统一成透明玻璃质感，而不是默认的纯色白底组件
3. 背景改为固定全屏的二次元图片或动图，不随页面滚动移动
4. 原生系统样式控件尽量替换为自定义 UI，避免破坏整体风格

---

## 3. 当前界面状态

### 3.1 页面风格

当前页面采用的是：

- 全屏固定二次元背景图
- 前景透明玻璃面板
- 浅描边 + 弱阴影
- 高亮选中态使用浅蓝粉渐变

整体目标不是厚重毛玻璃，而是“薄玻璃层覆盖在背景图之上”。

### 3.2 当前背景

当前背景图片文件：

- `bg-bocchi.png`

当前背景层在页面中的实现方式：

- [index.html](/E:/codex/guns/index.html)
- [styles.css](/E:/codex/guns/styles.css)

当前背景已经是：

- 固定全屏
- 不随滚动移动
- 使用 `background-image` 方式接入

如果后续要重新切回视频背景，可参考以前的 `bg-anime.mp4` 使用思路，但目前页面已经切换为图片背景方案。

## 4. 本轮已完成的优化内容

### 4.1 页面结构精简

已删除或弱化以下内容：

- 大块说明型标题
- 类似“直接查”“改枪码”这种过强的展示文案
- 开发阶段介绍型表达

保留原则：

- 用户一进站就能看到筛选、卡片、改枪码内容
- 靠界面结构本身表达“这是改枪码网站”

### 4.2 背景层重做

已完成：

- 背景从普通纯色改成固定全屏背景层
- 背景不跟随内容滚动
- 增加轻量覆盖层，防止前景完全丢失可读性
- 保留少量氛围光层用于增强画面融合感

### 4.3 玻璃质感统一

已完成统一处理的区域包括：

- 顶部数据区
- 筛选区
- 侧栏
- 改枪码结果卡片
- 标签
- 按钮
- 输入框
- 开关

当前玻璃风特点：

- 极低透明度背景
- 细描边
- 轻阴影
- 选中态高亮

### 4.4 去除 UI 模糊

之前前景层使用了模糊玻璃做法，但会影响背景人物展示。

当前已经改为：

- 去掉前景 UI 的 `backdrop-filter`
- 保留透明、描边、阴影

这样做的原因：

- 让背景图人物更清楚
- 保留透明层次感
- 避免组件区域把背景糊成一片

### 4.5 自定义下拉框

原生 `select` 已经替换为自定义下拉：

- 武器筛选
- 来源筛选

原因：

- 原生系统菜单无法稳定保持站内视觉风格
- 不同浏览器下会出现白底、深色系统菜单等不统一情况

自定义下拉当前支持：

- 点击展开/收起
- 点击外部关闭
- `Escape` 关闭
- 动态生成选项
- 选中态高亮
- hover 发光和描边

- [styles.css](/E:/codex/guns/styles.css)

## 6. 当前设计原则

后续继续优化时，建议保持以下原则：

### 6.1 背景优先级高

当前页面的核心视觉资产已经变成二次元背景图。

所以前景 UI 必须满足：

- 不要太厚
- 不要太白
- 不要强模糊
- 不要用大面积纯色遮住人物

### 6.2 组件统一

所有交互控件尽量保持同一种视觉语言：

- 同级透明度
- 同级描边
- 同级圆角
- 同级 hover 反馈

避免出现：

- 某些组件特别亮
- 某些组件像系统默认样式
- 某些组件厚重像实体卡片

### 6.3 结果区仍然要可读

即使继续增强背景露出程度，也要保证以下内容仍可快速识别：

- 枪械名
- 改装描述
- 改枪码
- 价格标签
- 来源标签
- 复制按钮

## 7. 后续可继续优化的方向

下面是后续最值得继续做的优化方向。

### 7.1 背景构图适配

当前最可能继续优化的是背景图人物与前景区块的关系。

建议可做：

- 调整 `.bg-image` 的 `background-position`
- 根据不同背景图切换不同的位置方案
- 让人物避开主要筛选区和结果区

如果背景人物在右侧，可尝试：

- `background-position: right center;`

如果背景人物在左侧，可尝试：

- `background-position: left center;`

### 7.2 前景布局适配不同背景

以后如果频繁换背景图，不建议只调透明度。

更合理的方式是：

- 根据背景主体位置调整内容区宽度
- 让主要文字和卡片避开人物脸部区域
- 必要时改变侧栏和结果区比例

### 7.3 统一 hover 动效

目前部分组件已有 hover 效果，但全站还可以继续统一。

建议后续统一这些组件：

- 按钮
- 预算标签
- 自定义下拉选项
- 结果卡片
- 侧栏卡片

可加入：

- 轻微上浮
- 细描边发光
- 顶部高光移动

### 7.4 结果卡片层次优化

当前卡片已经透明化，但后续仍可继续提升精致度。

建议方向：

- 强化枪械名和价格的第一视觉层级
- 让改枪码字体更像信息条
- 卡片 hover 时增加轻微发光边缘

### 7.5 主题切换能力

如果以后会频繁测试不同背景，建议增加主题变量管理。

可考虑把以下内容进一步抽成 CSS 变量：

- 面板透明度
- 边框透明度
- 按钮高亮色
- 文本亮度
- 背景覆盖层强度

这样更方便做：

- 图片背景版
- 视频背景版
- 明亮背景版
- 人物偏左版
- 人物偏右版

---

## 8. 后续修改时的建议顺序

以后继续优化时，建议按这个顺序改：

1. 先定背景图或背景视频
2. 再调背景位置和遮罩强度
3. 再调前景透明度
4. 再调布局位置
5. 最后统一 hover、选中态和细节动效

原因：

- 背景一变，前景透明度和布局往往都要跟着变
- 先抠细节再换背景，返工会很多

## 10. 当前结论

当前版本已经完成了从“普通静态数据页”向“二次元透明玻璃风工具站”的第一轮重构。

已形成的基础能力：

- 固定背景层
- 高透明前景
- 非模糊玻璃 UI
- 自定义下拉筛选
- 可继续快速测试不同背景素材

后续继续优化时，最值得优先做的是：

1. 背景人物位置适配
2. 全站 hover 和选中态统一
3. 卡片层次和信息密度优化
