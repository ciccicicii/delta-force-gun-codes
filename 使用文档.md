# 改枪码网站使用文档

## 1. 项目简介

这是一个基于静态 HTML、CSS、JavaScript 实现的改枪码展示网站。

当前特点：

- 无需安装 `node_modules`
- 页面直接读取本地 JSON 数据
- 支持搜索、筛选、来源过滤、价格区间过滤
- 支持一键复制改枪码

当前主要页面与脚本文件：

- `index.html`：页面结构
- `styles.css`：页面样式
- `app.js`：数据读取、筛选、渲染逻辑
- `三角洲改枪码_步枪_全网合并.json`：当前网站使用的数据文件

## 2. 启动方法

由于页面使用了 `fetch()` 读取 JSON 数据，不能直接双击 `index.html` 打开，必须通过本地 HTTP 服务启动。

### 方法一：使用 Python 启动

在项目目录下执行：

```powershell
cd E:\codex\guns
python -m http.server 8000
```

浏览器访问：

```text
http://localhost:8000
```

### 方法二：使用 VS Code Live Server

如果你使用 VS Code，可以：

1. 打开项目目录 `E:\codex\guns`
2. 安装 `Live Server` 插件
3. 右键 `index.html`
4. 选择 “Open with Live Server”

## 4. 数据来源与读取方式

当前前端脚本 [app.js](/E:/codex/guns/app.js) 中通过常量指定数据文件路径。

当前使用的是：

```js
const DATA_PATH = "./三角洲改枪码_步枪_全网合并.json";
```

页面启动后会自动读取该 JSON 文件，并将内容渲染到页面中。

## 5. JSON 数据格式要求

当前前端逻辑默认会读取以下字段：

- `枪械名称`
- `改枪码`
- `改装描述`
- `枪械价格`
- `来源`

推荐单条数据结构示例：

```json
{
  "枪械名称": "M4A1",
  "改枪码": "6FPBUPG06TRF107Q8PIO9",
  "改装描述": "中距离通用方案",
  "枪械价格": "25万",
  "来源": "aitags"
}
```

注意：

- `枪械价格` 当前会被前端提取数字部分用于价格筛选
- 如果价格为空或无法解析，会被归类为“价格待补”
- 如果 `枪械名称` 为“待识别”或类似值，会被归入未识别数据

## 7. 更新数据的方法

1. 保持 JSON 字段结构不变
2. 将新数据写入 `三角洲改枪码_步枪_全网合并.json`
3. 刷新网页

- 先备份原文件

## 9. 常见问题

### 1. 为什么双击 `index.html` 后看不到数据？

原因：

- 浏览器直接以 `file://` 方式打开时，`fetch()` 读取本地 JSON 往往会失败

解决：

- 使用本地 HTTP 服务启动，不要直接双击打开

### 2. 页面显示乱码怎么办？

当前仓库中部分页面文案和脚本字符串已经出现乱码，通常是文件编码处理不一致导致的。

排查建议：

- 确认文件保存为 `UTF-8`
- 确认编辑器没有用 GBK 或其他编码重新保存
- 优先检查 `index.html` 和 `app.js`

### 3. JSON 改了但页面没变化？

可能原因：

- 浏览器缓存未刷新
- 本地服务未指向当前目录
- JSON 格式错误导致前端读取失败

建议：

- 强制刷新页面
- 打开浏览器开发者工具查看报错
- 检查 JSON 是否为合法格式

## 10. 后续维护建议

如果你准备继续扩展这个网站，建议优先做这几件事：

1. 先修复页面乱码问题，统一编码为 `UTF-8`
2. 给 JSON 增加更稳定的字段规范
3. 把不同枪械分类拆分成独立数据文件
