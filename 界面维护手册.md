## 3. 换背景时改哪里

### 3.1 现在是图片背景

当前 HTML 背景结构在：

- [index.html](/E:/codex/guns/index.html)

你会看到：

```html
<div class="bg-scene" aria-hidden="true">
  <div class="bg-image"></div>
  ...
</div>
```

当前图片路径在：

- [styles.css](/E:/codex/guns/styles.css)

关键样式：

```css
.bg-image {
  background-image: url("./bg-bocchi.png");
}
```

### 3.2 如果要换成别的图片

只需要改：

- `.bg-image` 里的 `background-image`

例如：

```css
background-image: url("./你的新图片.png");
```

### 3.3 如果要切回视频背景

需要做两步：

1. 在 [index.html](/E:/codex/guns/index.html) 把 `.bg-image` 改回 `video`
2. 在 [styles.css](/E:/codex/guns/styles.css) 把 `.bg-image` 改回 `.bg-video`

推荐直接参考这类结构：

```html
<video class="bg-video" autoplay muted loop playsinline>
  <source src="./bg-anime.mp4" type="video/mp4">
</video>
```

## 4. 背景显示位置改哪里

如果背景人物挡住了内容，不要先急着调透明度，优先改背景位置。

### 4.1 图片背景位置

关键类：

- `.bg-image`

可改属性：

```css
background-position: center center;
```

常用改法：

- `left center`
- `center center`
- `right center`
- `60% center`
- `70% center`

例如：

```css
background-position: right center;
```

### 4.2 视频背景位置

如果以后换回视频，改的是：

```css
object-position: center center;
```

常用改法：

- `left center`
- `center center`
- `right center`
- `60% center`

---

## 5. 背景亮度和遮罩改哪里

### 5.1 背景亮度

图片或视频本体的明暗在下面改：

- `.bg-image`
- `.bg-video`

关键属性：

```css
filter: brightness(0.7) contrast(1.02) saturate(1.08);
```

如果人物太暗：

- 提高 `brightness`

例如：

```css
brightness(0.8)
```

如果背景太花：

- 稍微降低 `saturate`
- 或略微提高遮罩

### 5.2 背景遮罩

关键类：

- `.bg-video-overlay`

这是压在背景上方的一层。

如果背景人物看不清：

- 降低遮罩透明度

如果前景内容不够清楚：

- 稍微提高遮罩透明度

## 6. 前景透明度改哪里

这是后续最常改的一部分。

### 6.1 页面大容器透明度

下面这些类决定主要面板透明度：

- `.hero`
- `.section`
- `.panel`
- `.control-shell`
- `.result-card`
- `.hero-card`
- `.stat-card`
- `.empty-state`

这些类里一般都用了类似写法：

```css
background: linear-gradient(180deg, rgba(255, 255, 255, 0.022), rgba(255, 255, 255, 0.004));
```

规则：

- 前面的数值越大，越不透明
- 后面的数值越小，越接近完全透明

### 6.2 输入框和筛选透明度

重点类：

- `.search input`
- `.custom-select-trigger`
- `.toggle`
- `.tier-chip`

如果你想让筛选区更透，就改这些类的 `background`

### 6.3 下拉菜单透明度

重点类：

- `.custom-select-menu`
- `.custom-select-option`

展开菜单不透明度就在这里调。

## 7. 模糊效果改哪里

目前前景 UI 已经去掉了 `backdrop-filter`。

如果以后你又想试“更厚一点的玻璃感”，可以在这些类加回：

- `.hero`
- `.section`
- `.panel`
- `.result-card`
- `.control-shell`
- `.custom-select-menu`

例如：

```css
backdrop-filter: blur(12px);
-webkit-backdrop-filter: blur(12px);
```

但注意：

- 模糊一加，背景人物会更难看清

## 13. 常见修改场景

### 场景 1：背景人物被挡住

建议顺序：

1. 先改背景位置
2. 再降前景透明度
3. 最后再改布局宽度

不要一开始就疯狂降透明度。

### 场景 2：页面太透，看不清文字

建议顺序：

1. 稍微提高 `.hero`、`.section`、`.result-card` 的背景透明度
2. 提高 `--text`、`--muted`
3. 适当加一点背景遮罩

### 场景 3：下拉菜单不协调

优先改：

- `.custom-select-menu`
- `.custom-select-option`
- `.custom-select-option.active`

### 场景 4：按钮太突出或太弱

优先改：

- `.button-primary`
- `.button-secondary`
- `.copy-button`

---

## 14. 推荐修改顺序

以后每次继续优化，建议按下面顺序操作：

1. 确定背景素材
2. 调背景位置
3. 调背景亮度和遮罩
4. 调前景透明度
5. 调文字可读性
6. 最后调 hover、发光、选中态

这样返工最少。

---

## 15. 当前最值得继续做的事

如果下一步继续优化，我建议优先做这三件：

1. 针对当前 `bg-bocchi.png` 调整 `background-position`
2. 给结果卡和按钮统一 hover 发光语言
3. 让顶部区、筛选区、结果区根据背景人物位置重新分布
